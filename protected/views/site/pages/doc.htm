Pananda creating and maintaining challenges

This document describes the process of creating and managing challenges in Pananda. If there are any uncertainties, please contact us at info.pananda@gmail.com

Getting to the backend

The backend provides the interface to create and maintain challenges. It can be reached by logging in with your account at http://pananda.net/login

Challenge prize structure and workflow

There is some complexity in the structure of the challenges, as different challenges may need different prize structures, and each challenge needs to be manually validated.

The prize (prizes for winning pictures) structure can be set up in 3 ways:

-          The classic way. In the classic way, you hand pick the winners and award them with a prize. For example, the first winner wins a trip and the second winner $200.

-          The conditional way. This structure is added to be able to give each contestant who makes an effort a smaller prize. For example, each picture with more than 5 likes will get a 20% discount.

-          The classic-conditional way. This is a combination of the 2 options described above. So the winner might win 100$, while every photo with more than 10 likes will get 10% discount.

As we want to manually validate every challenge before it goes live, its important to note the following workflow:

-          (register and login)

-          Create a new challenge (see below for detailed explanation)

-          Save/modify/rethink the challenge

-          Request an admin validation at least 48 hours before the start date of the challenge

-          Modifying/deleting the challenge is not allowed now anymore

-          You will be notified at least 24 hours before the start if its accepted or declined

-          If its declined, modify the challenge based on the comments and request a new validation

-          If accepted, the challenge will be going live at the start date. The challenge cannot be modified or deleted when live. If there is an adjustment that needs to be done, please contact the admins.

-          During the challenge, if the conditional or classic-conditional prize structure has been chosen, users will automatically be emailed a security code when they pass the condition. The backend will generate a list with these codes, which can be found by going to the backend, going to the challenge and clicking conditional winners list.
For example, Alex exceeds the condition of 5 likes. Alex will receive an email with the code TESTCODE. When claiming his prize, Alex identity can be validated by looking up his email address in the mentioned list. If the codes match, the user is Alex and the prize can be redeemed. The code can now either be removed from the list (if its a one time only code) or left in place, if the prize can be handed out multiple times.

-          If chosen for the classic or classic-condition prize and the challenge ended, you get 7 days (after the set deadline has ended) to pick the winners. After picking the winners, a separate message can be composed for each winner. This way you will be able to tell them how to redeem their prize and what specific prize they won.

-          Prizes are directly transferred between the winner(s) and the company, without intervention of Pananda.

Creating a new challenge

Creating a new challenge is quite easy:

-          Login and go to challenges (top menu)

-          Click create challenge

-          Here you can create a challenge.

List of variables:

Image: the image thats the main attention around the challenge. Its the image that should challenge the user to engage.

Summary: 40 characters max, catching sentence

Description above fold: 140 char max description that will be displayed above the fold

Description Under: longer description that will be displayed when users click read more.

Note: the description should clearly contain the prizes and how these prized can be won

(For example: Prizes: #1 500$, #2, 100$. And every photo with more than 5 likes will get a 10% off their next purchase!)

Start date: Start date of the challenge (should be more than 24 hours ahead). All times are GMT+1.

Deadline: deadline (end date) of the challenge. Startdate and deadline should atleast be 24 h apart.

Tags: separated by a comma (,), with a maximum of 3 tags. These 3 tags should describe the target of the challenge. So instead of choosing tags like win, holiday and cash, try tags like happiness, goodlife or dark (depending on your intentions).

Prize structure type: see Challenge prize structure and workflow for a detailed description

Condition prize selector: what the condition is. For now, its only likes you can chose.

Condition prize variable: how much (in this case likes) someone should have before getting a prize. Should at least be 1. Choose this parameter wisely, too low and people will not put effort into sharing the photo of your products, to high and users will hardly reach it. We recommend somewhere between 5 and 10 for now.

Message to condition winner: We send an email to users who met the above condition with their verification code. The message you put here will be added to that email. This way you can congratulate them yourself. Note that this message should state what they won and how and where they can collect it!.
